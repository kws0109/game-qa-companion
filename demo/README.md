# 데모

나이트 크로우(MMORPG) PC 클라이언트 화면에서 inspect를 돌린 결과. 모두 LLM 라벨링(Claude 구독 연동)을 켜고 실행했다.

## 1. LLM 정리 — 정리 전

![정리 전](01_llm_cleanup_before.png)

캐릭터 선택 화면. CV 검출 + 안정성 마스크 단계에서 161개 후보가 잡혔다. 캐릭터 모델·배경 텍스처가 다수 섞여 있다.

## 2. LLM 정리 — 정리 후

![정리 후](02_llm_cleanup_after.png)

같은 화면에 LLM 라벨링을 적용한 결과. 비UI로 판정된 164개를 자동 제거하고 실제 UI 8개만 남았다. 각 요소에 역할·이름이 부여돼 박스 위에 표기된다 — 뒤로가기·시작하기·삭제하기 버튼, 화면 제목, 좌측 상단 정보 패널 등.

## 3. 복잡한 인게임 화면

![인게임 필드](03_complex_field_detection.png)

인게임 필드. 미니맵·상단 바·버프 아이콘·스킬 슬롯·HP/스탯 등 UI 82개를 식별했다. 단 들판 텍스처에 작은 오탐 박스가 일부 남는다 — 캐릭터 선택 화면(1·2)에서는 비UI가 명확해 LLM이 거의 다 걸러내지만, 필드는 작은 텍스처 조각을 아이콘과 구분하기 어렵다. 검출 단계(엣지 임계·안정성 마스크·필터)에서 오탐률을 줄이는 방법을 계속 실험 중이며, 현재는 vision+LLM 조합의 방향성이 맞는지 검증하는 프로토타입 단계다.
